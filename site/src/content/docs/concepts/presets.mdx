Not every player needs to support every use case. To address this, Video.js has **features** -- self-contained units of player functionality. For example, the "playback" feature adds `paused` and `play()`, "volume" adds `volume` and `setVolume()`, and so on. More on features, <DocsLink slug="concepts/features">later</DocsLink>.

A **preset** is a curated set of features bundled for a specific use case. Instead of wiring up individual features one by one, you pick a preset that matches what you're building. For example, the "video" preset includes features needed for a general website video player:

<FrameworkCase frameworks={["react"]}>
```tsx title="App.tsx"
import { createPlayer } from '@videojs/react';
import { videoFeatures, Video, VideoSkin } from '@videojs/react/video';

const Player = createPlayer({ features: videoFeatures });

function App() {
  return (
    <Player.Provider>
      <VideoSkin>
        <Video src="video.mp4" />
      </VideoSkin>
    </Player.Provider>
  );
}
```
</FrameworkCase>
<FrameworkCase frameworks={["html"]}>
```html title="index.html"
<script type="module">
  import '@videojs/html/video/player';
  import '@videojs/html/video/skin';
  import '@videojs/html/video/skin.css';
</script>

<video-player>
  <video-skin>
    <video slot="media" src="video.mp4"></video>
  </video-skin>
</video-player>
```
</FrameworkCase>

## Available presets

The default presets are simply "Video" and "Audio". These cover a baseline set of features, roughly the set of controls you'd expect from the HTML `<video>` and `<audio>` tags. Beyond the defaults, presets target more specific use cases. The `background-video` preset, for example, needs layout but not controls. Over time we'll add more, for example creator-platform players, short-form swipe-up players, TV app streaming players. On the audio side, a podcast player and a music player have a different set of needs. Presets give these use cases their own tailored feature sets.

<FrameworkCase frameworks={["react"]}>

| Preset | Import | Use case |
|--------|--------|----------|
| **Video** | `@videojs/react/video` | General website video |
| **Audio** | `@videojs/react/audio` | Audio player |
| **Background video** | `@videojs/react/background` | Hero/background video — layout only, no controls |

</FrameworkCase>

## How presets work

A preset is just an array of <DocsLink slug="concepts/features">features</DocsLink>. When you pass a preset to `createPlayer`, it creates a player with those features enabled.

```tsx title="App.tsx"
import { createPlayer } from '@videojs/react';
import { videoFeatures } from '@videojs/react/video';

// videoFeatures includes playback, volume, time, fullscreen, etc.
const Player = createPlayer({ features: videoFeatures });
// Player.Provider now has all video-related state and actions

// Spread the preset to extend it
const ExtendedPlayer = createPlayer({ features: [...videoFeatures, customFeature] });
```
</FrameworkCase>

## Extending presets
By default, you'll be using the element that comes with a preset:
```html
<script type="module">
  import '@videojs/html/background/player';
  import '@videojs/html/background/skin';
  import '@videojs/html/background/skin.css';
</script>
<background-video-player>
  <background-video-skin>
    <video slot="media" src="video.mp4"></video>
  </background-video-skin>
</background-video-player>
```

However, you might want to extend that functionality. For example, if you wanted to add a play/pause button to your background video:
```html title="index.html"
<script type="module">
  import { createPlayer, playback, MediaElement } from '@videojs/html';
  import { backgroundFeatures } from '@videojs/html/background';
  import '@videojs/html/media/container';
  import '@videojs/html/ui/play-button';

  // Spread the preset and add the playback feature
  const { ProviderMixin } = createPlayer({
    features: [...backgroundFeatures, playback],
  });

  class MyPlayer extends ProviderMixin(MediaElement) {
    static readonly tagName = 'my-player';
  }

  customElements.define('my-player', MyPlayer);
</script>

<my-player>
  <media-container>
    <video slot="media" src="video.mp4" autoplay muted loop playsinline></video>
    <media-play-button>Play / Pause</media-play-button>
  </media-container>
</my-player>
```
</FrameworkCase>

## Presets and skins

<DocsLink slug="concepts/skins">Skins</DocsLink> are designed for specific presets. Each skin expects the features its preset provides. For example, a video skin uses fullscreen and picture-in-picture controls, while an audio skin doesn't.

<FrameworkCase frameworks={["react"]}>

| Preset | Available skins |
|--------|-----------------|
| `videoFeatures` | `<VideoSkin>`, `<MinimalVideoSkin>` |
| `audioFeatures` | `<AudioSkin>`, `<MinimalAudioSkin>` |
| `backgroundFeatures` | `<BackgroundVideoSkin>` |

Each preset's export includes its skins:

```tsx
// Includes videoFeatures, Video, VideoSkin, MinimalVideoSkin
import { videoFeatures, Video, VideoSkin } from '@videojs/react/video';
```

</FrameworkCase>

| `<audio-player>` | `<audio-skin>`, `<audio-minimal-skin>` |
| `<background-video-player>` | `<background-video-skin>` |

</FrameworkCase>
